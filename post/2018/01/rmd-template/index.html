Words of Warning

  1. Never write index.md or index.Rmd. Always use _index.Rmd.
    (Per Issue #84 · digitalcraftsman/hugo-material-docs)
  2. Never write styles for .header, only header.header.
  3. Always check twice and preview the .html page.
  4. Always run blogdown::hugo_build() before pushing to git, otherwise baseURL is not fully implemented.
    Source: D.3 Building a website for local preview | blogdown: Creating Websites with R Markdown

When writing maths expressions,

Do this:
$\LaTeX$

Not this:
$ \LaTeX $

Display Code

R Code Block

Type

​```{r}
    (your code here)
​```

All codes will be displayed but not executed under default settings of this template. To turn on executing codes, use the chunk option:

```{r, eval = TRUE}
    (your code here)
```

Code of Other Languages

Type:

​```js
    (your code here)
​```

If blogdown is not properly highlighting your code, add the language name to config.yaml according to highlight.js/SUPPORTED_LANGUAGES.md.

params:
  highlightjsVersion: 10.0.0
  highlightjsCDN: //cdnjs.cloudflare.com/ajax/libs
  highlightjsLang: ["r", "applescript", "python", "vim"] #<<
  highlightjsTheme: "atom-one-light"

Plot

In order to refer to a figure, you must name the chunk that produces it, and turn on eval.
See Figure 1 for example:

par( mar = c( 0, 1, 0, 1 ))
pie(
  c( 280, 60, 20 ),
  c( 'Sky', 'Sunny side of pyramid', 'Shady side of pyramid' ),
  col = c( '#0292D8', '#F7EA39', '#C4B632' ),
  init.angle = -50, border = NA
)
